## Introducing Content Quality Audit — for SEO and the AI era

Today we're shipping the Content Quality Audit — a way to check whether your writing actually meets the bar Google (and AI engines) hold good content to.

Most SEO checklists obsess over technical issues: schema markup, page speed, heading tags. If you have a halfway decent blog template, those are already handled. What's actually hard — and what separates ranking content from invisible content — is the writing itself. So we built a ruleset around that.

The audit runs your content through seven dimensions grounded in E-E-A-T and extended for the GEO era: **Clarity, Organization, Rigor, Exclusivity, Experience, Expertise, and Trust**. You get a scored report with specific findings — which claims need data, where the experience signals are missing, whether the core answer is buried.

Available as an MCP tool for your AI agents and inside the RankEarly dashboard. Run it on a URL or paste in a draft directly.

## Keyword expansion gets flexible

Sometimes you know exactly which keywords to expand. Sometimes you just have a vague topic idea. The new Keyword Expansion handles both.

**Two ways in:**

Describe a topic in plain English and we'll generate starting keywords for you. Or skip that step entirely and paste your own list. Useful if you've already done research elsewhere or just prefer full control.

**Effort levels, not keyword counts:**

Pick how broad the search should be. Ultra Low focuses tight on one term. Ultra High sweeps an entire category. You'll see the estimated keyword range upfront.

**Pricing is now two parts:**

A fixed search cost based on effort level, plus 1 credit per keyword returned.

<Update label="March 28, 2026">

## Keyword Library: a shared foundation for all your research

Research in RankEarly now runs on a **keyword library** — a persistent, reusable database of keywords and search data for a specific market.

Previously, each research session held its own keyword set. That worked for simple UI workflows, but made it hard to build up research over time or run reliable AI agent workflows. Now everything runs on a shared library that grows with every expansion run and every SERP you inspect.

A few things that change for the better:

- **One library per market.** Each library is scoped to a country and language. Run multiple libraries if you target multiple markets — the keyword sets are independent, so your content stays genuinely tailored rather than just translated.

- **Research compounds.** Run keyword expansion multiple times on the same library. Keywords you inspect during SERP analysis get saved automatically. The more you research, the richer your library becomes.

- **Content pillars that evolve.** After you've gathered enough keywords, run Topic Clustering to turn your library into a structured content plan. Add or remove topics as your business changes — the library persists, so you're always building on what you already have.

- **Cleaner AI agent workflows.** Agents now work against a named, deterministic context. No more ambiguity about which keywords belong where.
