<Update label="March 28, 2026">

## Keyword Library: a shared foundation for all your research

Research in RankEarly now runs on a **keyword library** — a persistent, reusable database of keywords and search data for a specific market.

Previously, each research session held its own keyword set. That worked for simple UI workflows, but made it hard to build up research over time or run reliable AI agent workflows. Now everything runs on a shared library that grows with every expansion run and every SERP you inspect.

A few things that change for the better:

- **One library per market.** Each library is scoped to a country and language. Run multiple libraries if you target multiple markets — the keyword sets are independent, so your content stays genuinely tailored rather than just translated.

- **Research compounds.** Run keyword expansion multiple times on the same library. Keywords you inspect during SERP analysis get saved automatically. The more you research, the richer your library becomes.

- **Content pillars that evolve.** After you've gathered enough keywords, run Topic Clusters to turn your library into a structured content plan. Add or remove pillars as your business changes — the library persists, so you're always building on what you already have.

- **Cleaner AI agent workflows.** Agents now work against a named, deterministic context. No more ambiguity about which keywords belong where.

<Update label="March 21, 2026">

## RankEarly now works with your AI agents

You can now run RankEarly's keyword research and SERP analysis directly from Claude Code, Cursor, or any MCP-compatible AI agent — no browser tab required.

We repackaged our core SEO algorithms as [RankEarly MCP tools](https://docs.rankearly.pro/mcp-server) and agent skills. Add the MCP server to your config, authorize with OAuth, and your agent can call keyword expansion, SERP data, and clustering tools directly. Context stays focused on your actual task.

[Read the full announcement →](https://www.rankearly.pro/blog/rankearly-now-works-with-your-ai-agents)

</Update>

<Update label="March 16, 2026">

## RankEarly MCP is coming

We switched our auth provider from Logto to Better Auth. The main reason: Better Auth supports dynamic client registration, which is required for compliant MCP OAuth flows — and it's what we need to ship RankEarly MCP.

Nothing changes for you today. Sign-in works the same, your data is untouched. More on RankEarly MCP soon.

</Update>
