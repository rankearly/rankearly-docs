A keyword library is a shared, reusable database of keywords and search data scoped to a specific market — defined by country and language. Every RankEarly research run reads from and writes to a library, giving all your workflows a consistent and growing foundation.

## What a keyword library contains

- **Keywords** — expanded keywords with search volume and difficulty data
- **SERP snapshots** — captured search results from keyword expansion and SERP gap analysis
- **Topic pillars** — content pillars generated by [Topic Clusters](/research/topic-clusters) from the library's keywords

## One library per market

A keyword library is defined by two attributes:

| Attribute | What it means |
|---|---|
| **Country** | The geographic market the keywords are based on |
| **Language** | The language the keywords are expressed in |

If you are targeting multiple markets — say, the US in English and Germany in German — create a separate library for each. RankEarly expands keywords independently for each market, so the results may be quite different even when the seed idea is the same.

This means you can publish tailored content for each market rather than running the same articles through a translator.

## Multiple expansion runs on one library

You can run [Keyword Expansion](/research/keyword-expansion) as many times as you want on the same library. Each run adds more keywords and SERP data to the library without replacing what was already there.

A good workflow:
1. Run keyword expansion for your core topic
2. Run it again for a related angle or competitor product
3. Keep expanding until you have broad coverage
4. Then run [Topic Clusters](/research/topic-clusters) to organize everything into content pillars

Keywords you inspect during [SERP gap analysis](/research/primary-term) are also automatically saved back into the library.

## Content pillars from your library

Once you have built up a keyword library, you can run [Topic Clusters](/research/topic-clusters) to organize those keywords into content pillars. Each pillar groups related subtopics under one subject — giving you a structured content plan based on actual search data.

After the initial clustering, you can:
- **Enrich pillars** — add new pillars as your product or market expands
- **Remove pillars** — drop topics that are no longer a priority
- **Re-cluster** — run Topic Clusters again to surface new pillars as your library grows

The library persists across runs, so your content architecture evolves alongside your research instead of starting from scratch each time.

## Managing your libraries

You can create and manage keyword libraries from the RankEarly dashboard or via the [MCP server](/mcp-server) if you're using an AI agent workflow.

| Action | How |
|---|---|
| Create a library | Dashboard → your project → Keyword Library → New library |
| Select a library for research | Choose it when starting a Keyword Expansion or Topic Clusters run |
| View library keywords | Dashboard → Keyword Library → select a library |
| View pillars | Dashboard → Keyword Library → select a library → Pillars tab |

Keyword libraries act as a cloud persistence layer for your keyword data. When an agent runs [RankEarly skills](https://www.rankearly.pro/skills) to expand keywords or analyze SERPs, the results land in a named library that any team member — or another agent — can build on later. Downstream research like topic clustering or content briefs can draw from it directly. Your marketing team shares one growing knowledge base.

Keywords are also exportable as CSV if you need them outside RankEarly.
