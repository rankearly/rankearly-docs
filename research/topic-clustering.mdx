---
title: "Topic Clustering"
description: "Turn an existing keyword library into keyword clusters and broader topics you can use as a topical map."
---

Topic Clustering turns an existing keyword library into a topical map. It groups keywords into keyword clusters based on SERP similarity, then uses an NLP clustering step to group those keyword clusters into broader topics.

Each keyword cluster is a group of keywords that share a similar SERP. Each topic is a broader theme made up of one or more keyword clusters.

## When to use it

- You want a topical map, not a single article idea
- Your project already has a keyword library and you want to turn it into topics
- You want to understand which themes are most worth building content around

## Before you start

Before you run it:

- Choose a keyword library that belongs to the same project
- Make sure the library already contains keywords
- Use [Keyword Expansion](/research/keyword-expansion) first if the project does not have a usable library yet

## How it works

<Steps>
  <Step title="Start from an existing keyword library">
    Topic Clustering uses the keywords already stored in your keyword library.
  </Step>
  <Step title="Cluster related keywords">
    RankEarly compares the SERPs for keywords in that library and groups together keywords whose search results materially overlap. That forms your keyword clusters. The label for each keyword cluster comes from the highest-volume keyword inside that cluster.

    <img
      src="https://cdn.rankearly.pro/images/docs/serp-clustering.jpg"
      alt="Diagram showing keywords grouped by overlapping search results into keyword clusters, with the highest-volume keyword used as each cluster label"
      width="920"
    />
  </Step>
  <Step title="Organize clusters into topics">
    RankEarly then uses an NLP algorithm to group related keyword clusters into broader topics. This step is conservative by design: it prefers leaving weak matches separate instead of forcing them into the same topic.

    <img
      src="https://cdn.rankearly.pro/images/docs/topic-clustering.jpg"
      alt="Diagram showing keyword clusters grouped by NLP similarity into broader topics"
      width="920"
    />
  </Step>
  <Step title="Prioritize what to write">
    Review the returned topics and keyword clusters to decide which areas best fit your project, your audience, and the places where you can publish credible experience-backed content.
  </Step>
</Steps>

## Reading the results

Review each topic as a group of related keyword clusters. Review each keyword cluster as a group of closely related searches.

Topic names come from the most central keyword cluster inside that topic. Treat them as working labels, not final page titles.

Use the keyword cluster metrics and underlying keyword clusters to decide:

- which topics best align with your project
- which topics best align with your project and expertise
- which keyword clusters are the best places to start
- which opportunities should move into a [content brief](/planning/content-brief)

## Where topics are saved

Topics are saved to the keyword library. When you run Topic Clustering again for the same library, the new run replaces the previous topics for that library.

## What to do with the results

- Save the topics that match your product, audience, and expertise
- Start with keyword clusters that combine lower KD with stronger search volume
- Create a [content brief](/planning/content-brief) from the keyword clusters you plan to write next

## Tips

<AccordionGroup>
  <Accordion title="Use the KD filter for cleaner early-stage plans">
    The filter is on by default. It removes keywords above `KD 70`, which usually produces a more realistic topical map for newer sites. Turn it off when you want the full market view or your site can compete on harder terms.
  </Accordion>
  <Accordion title="Topic Clustering depends on your library quality">
    Topic Clustering reuses the keywords and SERP snapshots already saved in your library. If the library is thin or outdated, the topical map will be thin too.
  </Accordion>
  <Accordion title="Build the library from multiple angles before clustering">
    Topic Clustering works best when the library already covers different angles of the same market. Run Keyword Expansion across features, audiences, and business angles first, then run Topic Clustering on that fuller library.
  </Accordion>
  <Accordion title="Re-running replaces the previous topics">
    Topic Clustering does not keep multiple topic sets for the same library. If you want a different topical map, duplicate or create a separate library first.
  </Accordion>
</AccordionGroup>
