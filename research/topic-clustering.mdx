Topic Clustering groups the keyword clusters already in your library into a small set of broader themes — the hub layer above your page-level topics.

In RankEarly, a **topic** means a keyword cluster — a page-level opportunity. A **theme** is a broader subject area that organizes related topics into coherent information architecture.

The delivered structure is **Theme → Keyword Clusters**. Each keyword cluster is the minimum unit for content planning, and one page should map to exactly one keyword cluster.

## When to use it

Run Topic Clustering when:

- the library has grown large and you want a navigable map grouped by broader subject
- you want a hub-level view to plan a [pillar-cluster](/glossary#pillar-cluster-model) architecture

For a small or focused library, working directly from the keyword clusters in your [library](/keyword-library) is usually enough.

## Before you start

- Choose a keyword library that belongs to the same project
- Make sure the library already contains keywords with SERP snapshots — the existing keyword clusters are the input
- Use [Keyword Expansion](/research/keyword-expansion) first if the project does not have a usable library yet

## Reading the results

Review each theme as a group of related keyword clusters. Review each keyword cluster as one page opportunity — a group of closely related searches Google answers with the same results.

Theme names come from the most central keyword cluster inside that theme. Treat them as working labels, not final page titles.

Use the cluster metrics and underlying keywords to decide:

- which themes best align with your project and expertise
- which keyword clusters are the best places to start writing
- which opportunities should move into a [content brief](/planning/content-brief)

## One page, one keyword cluster

A keyword cluster is the minimum unit for content planning. Writing multiple pages for the same keyword cluster creates cannibalization — Google already treats those queries as the same question, so two of your pages end up competing with each other.

Plan one page per keyword cluster. If a cluster feels too narrow to fill a page, it usually belongs inside a broader cluster; re-check the clusters in your library before splitting.

## Relationship to the pillar-cluster model

Topic Clustering is the organization step. It produces the grouped structure: themes at the top, keyword clusters underneath.

The [pillar-cluster model](/glossary#pillar-cluster-model) is a downstream content-architecture strategy you can build from that output:

- use a theme as the hub (the pillar page covers the broad subject)
- use the keyword clusters under that theme as the supporting pages
- interlink them so search engines see the topical structure

Topic Clustering gives you the map. The pillar-cluster model is one way to turn that map into a published site architecture.

## Where themes are saved

Themes are saved to the keyword library. Re-running Topic Clustering on the same library replaces the previous themes; keyword clusters are not affected.

## What to do with the results

- Keep the themes that match your product, audience, and expertise
- Start with keyword clusters that combine lower KD with stronger search volume
- Create a [content brief](/planning/content-brief) from the keyword clusters you plan to write next
- Use the themes to plan pillar pages once you have enough supporting cluster content

## Tips

<AccordionGroup>
  <Accordion title="Topic Clustering depends on your library quality">
    Themes are only as good as the clusters they group. If the library is thin or outdated, the topical map will be thin too.
  </Accordion>
  <Accordion title="Build the library from multiple angles before clustering">
    Themes are strongest when the library already covers different angles of the same market. Run Keyword Expansion across features, audiences, and business angles first, then run Topic Clustering on that fuller library.
  </Accordion>
  <Accordion title="Semantic grouping, not linguistic similarity">
    Themes reflect semantic relatedness between page-level topics. Phrases that look similar but answer different questions — for example, `what is CRM` and `what is CRM account` — will not be forced into the same theme.
  </Accordion>
  <Accordion title="Re-running replaces the previous themes">
    Topic Clustering does not keep multiple theme sets for the same library. If you want a different topical map, duplicate or create a separate library first.
  </Accordion>
</AccordionGroup>
