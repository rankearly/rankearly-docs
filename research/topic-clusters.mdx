title: "Topic clusters"
description: "Turn an existing keyword library into topics and keyword clusters you can prioritize for your content plan."
---

Topic Clusters turns an existing keyword library into a structured content plan. It helps you analyze the keyword landscape around your project, group related search intents into article-sized keyword clusters, and prioritize the topics that best fit what you want to rank for.

Each keyword cluster is one article opportunity. Each topic is the broader theme those articles support.

## When to use it

- You want a full content plan, not a single article idea
- Your project already has a keyword library and you want to turn it into topics
- You want to understand which themes are most worth building content around

## Before you start

Before you run it:

- Choose a keyword library that belongs to the same project
- Make sure the library already contains keywords and saved SERP snapshots
- Use [Keyword Expansion](/research/keyword-expansion) first if the project does not have a usable library yet

## How it works

<Steps>
  <Step title="Start from an existing keyword library">
    Topic Clusters uses the keywords already stored in your keyword library instead of starting from scratch.
  </Step>
  <Step title="Cluster related keywords">
    RankEarly compares the stored SERP snapshots in that library and groups related keywords into clusters. Each cluster becomes one keyword cluster, which usually maps to one article to write.
  </Step>
  <Step title="Organize clusters into topics">
    AI reviews the keyword clusters and groups the related keyword clusters into topics.
  </Step>
  <Step title="Prioritize what to write">
    Review the returned topics and their keyword clusters to decide which themes best fit your project, your audience, and the areas where you can publish credible experience-backed content.
  </Step>
</Steps>

## Reading the results

Review each topic as a theme and each keyword cluster as an article opportunity within that theme.

Use the keyword cluster metrics and underlying keyword clusters to decide:

- which topics best align with your project
- which topics best align with your project and expertise
- which keyword clusters are the best first articles to write
- which opportunities should move into a [content brief](/planning/content-brief)

## Saved topics and research history

You can save the topics you want to keep working from and revisit past Topic Clusters runs through research history later.

## What to do with the results

- Save the topics that match your product, audience, and expertise
- Start with keyword clusters that combine lower KD with stronger search volume
- Create a [content brief](/planning/content-brief) from the keyword clusters you plan to write next
- Run Topic Clusters separately for different features or verticals if one library covers multiple themes

## Tips

<AccordionGroup>
  <Accordion title="Use the KD filter for cleaner early-stage plans">
    The filter is on by default. It removes keywords above `KD 70`, which usually produces a more realistic plan for newer sites. Turn it off when you want the full market view or your site can compete on harder terms.
  </Accordion>
  <Accordion title="Topic Clusters depends on your library quality">
    Topic Clusters reuses the keywords and SERP snapshots already saved in your library. If the library is thin or outdated, the topics will be thin too.
  </Accordion>
  <Accordion title="Run it per feature or vertical">
    If one project covers several features, industries, or audiences, run Topic Clusters on a focused library for each one. That keeps the topics easier to name, save, and brief.
  </Accordion>
</AccordionGroup>
